Правила работы с репозиторием GitHub

Каждый из участников команды имеет отдельную ветку в которой хранятся разрабатываемые им решения, файлы и т.д. Главный разработчик также ведёт основную ветку для разработки - "Development", которая обновляется при помощи слияния веток. По мере надобности и выполнения задач, главный разработчик отправляет файлы в ветку "Stable" , в случае, если ошибки были обнаружены, он отправляет файлы с найденными ошибками ответственным за создание этого файла. После исправления ошибок участник команды вновь отправляет файлы в ветку "Development" , и цикл повторяется до тех пор, пока не будет получено полностью готовое и исправное решение. Ответственный за документацию имеет доступ и обращается к веткам "Development" и "Stable".
